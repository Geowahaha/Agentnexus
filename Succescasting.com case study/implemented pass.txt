Risks / notes

• Sitemap does not carry Content-Signal (XML route; not in header rules). Low impact.
• Cloudflare HTML cache rule can still serve stale pages up to ~4h if purge fails — post-deploy purge ran successfully.
• WebMCP server hints are in a sr-only form; the interactive AI chat widget is client-rendered and won’t appear in no-JS crawls.
• Audit skill pack extracted to scripts/audit-ai-visibility-pack/ for future runs.

2. Agent discoverability — new agents.txt, richer llms.txt (27 links), unified policy files.
3. Content-Signal header — explicit opt-out of training while allowing search/citation grounding.
4. On-page SEO — single stable H1, /products H1 for crawlers, product pages already have H1 + schema + RFQ CTAs.
5. WebMCP hints — minor agentic UX signal; unscored by audit.

Phrase as evidence-weighted, not guaranteed causality.
